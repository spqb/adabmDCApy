    Training DCA models üöÄ

    All versions of adabmDCA ‚Äî Python, Julia, and C++ ‚Äî expose the same command-line interface through the adabmDCA command.

    To see the complete list of training options:

    $ adabmDCA train -h

    The standard command to start training a DCA model is:

    $ adabmDCA train -m <model> -d <fasta_file> -o <output_folder> -l <label>

    Arguments üß©

    • <model> ‚àà {bmDCA, eaDCA, edDCA}
      Selects the training routine.
      By default, the fully connected bmDCA algorithm is used. edDCA can follow two different routines: either it decimates a pre-trained bmDCA model, or it first trains a bmDCA model and then decimates it.
    • <fasta_file> ‚Äì Path to the FASTA file containing the training MSA.
    • <output_folder> ‚Äì Folder where results will be stored (created if missing).
    • <label> ‚Äì Optional tag for output files.

    Training Behavior ‚öôÔ∏è

    Training stops when the Pearson correlation between model and empirical connected correlations reaches the target value (default: 0.95).

    • Early training is fast (e.g., Pearson ‚âà 0.9 after ~100 iterations).
    • Approaching higher values takes significantly longer (power‚Äëlaw decay).

    For a quick coarse model, set:

    --target 0.9

    Output Files üìÅ

    During training, adabmDCA maintains three output files:

    • <label>_params.dat ‚Äì Non‚Äëzero model parameters
    • Lines starting with J ‚Üí couplings

    • Lines with h ‚Üí biases

    • <label>_chains.fasta ‚Äì State of the Markov chains

    • <label>_adabmDCA.log ‚Äì Log file updated throughout training

    Update intervals: - bmDCA: every 50 updates
    - eaDCA, edDCA: every 10 updates

    Restoring Interrupted Training üîÑ

    Resume training using:

    $ adabmDCA train [...] -p <file_params> -c <file_chains>

    Importance Weights üèãÔ∏è‚Äç‚ôÇÔ∏è

    Provide custom weights with:

    --weights <path>

    Otherwise, weights are computed automatically and stored as:

    <label>_weights.dat

    Options:

    • --clustering_seqid <value> ‚Äì default: 0.8
    • --no_reweighting ‚Äì use uniform weights

    Choosing the Alphabet üî†

    Default alphabet: protein.

    Specify alternatives:

    • RNA ‚Üí --alphabet rna
    • DNA ‚Üí --alphabet dna
    • Custom ‚Üí
      --alphabet ABCD-

    eaDCA üå±

    Enable with:

    --model eaDCA

    Key hyperparameters:

    • --factivate ‚Äì fraction of inactive couplings activated (default: 0.001)
    • --gsteps ‚Äì parameter updates per graph update (default: 10)

    Recommended: reduce sweeps to 5.

    edDCA ‚úÇÔ∏è (Decimated DCA)

    Run decimation:

    $ adabmDCA train -m edDCA -d <fasta_file> -p <params> -c <chains>

    Two workflows:

    1. Use pre‚Äëtrained bmDCA (params + chains)
    2. Train bmDCA automatically, then decimate

    Key hyperparameters:

    • --gsteps ‚Äì default: 10
    • --drate ‚Äì pruning fraction (default: 0.01)
    • --density ‚Äì target graph density (default: 0.02)
    • --target ‚Äì Pearson threshold (default: 0.95)

    Choosing Hyperparameters üéöÔ∏è

    Defaults work well for clean and moderately diverse MSAs. For more difficult datasets, consider tuning:

    Learning Rate

    • Default: 0.01
    • If chains mix poorly, try:
      --lr 0.005

    Number of Markov Chains

    • Default: 10,000
    • Using fewer chains reduces the memory required to train the model, but it may also lead to a longer algorithm convergence time.
    • Change with:
      --nchains <value>

    Number of Monte Carlo Steps

    • Controlled by --nsweeps
    • Default: 10
    • Recommended range: 10‚Äì50. Higher values drastically increase the training time and, empirically, do not help much the model convergence.

    Regularization (Pseudocount)

    Controlled by --pseudocount.

    Default:

    Œ± = 1 / M_eff

    Increasing Œ± (e.g. Œ± = 0.001 or 0.01) may help when the training struggle converging or the mixing time of the model is very high, but it also makes the model less expressive.
